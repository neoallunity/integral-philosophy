# TeX-политика Журнала

Этот документ определяет технические стандарты и соглашения для подготовки статей с использованием TeX/LaTeX. Цель этой политики — обеспечить семантическую целостность, воспроизводимость, архивируемость и возможность легкого преобразования в другие форматы (JATS XML, HTML, ePub).

## 1. Основные принципы

- **Структура и семантика**: Статьи должны быть структурированы семантически, а не визуально. Используйте стандартные команды LaTeX для разделов, списков, таблиц и т.д. (`\section`, `\item`, `\caption`).
- **Воспроизводимость**: Сборка любой статьи должна быть полностью автоматизирована и не зависеть от локальных настроек автора.
- **Разделение данных и оформления**: Содержимое статьи (текст, данные) должно быть отделено от ее визуального представления.

## 3. Структура проекта

- **Каталог для каждой статьи**: Каждая статья размещается в собственном подкаталоге внутри `articles/`. Например, `articles/ivanov/`.
- **Содержимое каталога статьи**:
  - `main.tex`: Основной файл статьи.
  - `references.bib`: Файл библиографии в формате BibLaTeX.
  - `images/`: Подкаталог для изображений, используемых в статье.

## 4. Библиография

- **Формат**: Источники должны быть представлены в машиночитаемом формате (`.bib`).
- **Инструменты**: Для управления библиографией используется связка **BibLaTeX + Biber**. Использование BibTeX запрещено.
- **Стиль цитирования**: Применяется единый стиль цитирования, определённый в классе журнала.

## 5. Класс и пакеты

- **Класс журнала**: Все статьи должны использовать предоставляемый класс журнала `journal.cls`. Запрещается изменять этот файл.
- **Допустимые пакеты**: Список разрешенных пакетов фиксирован и подключается централизованно через преамбулу. Включение произвольных пакетов авторами запрещено.
  - *Примерный список разрешенных пакетов будет добавлен позже.*
- **Запрещенные пакеты**: Запрещается использование пакетов, которые конфликтуют с основной конфигурацией, являются устаревшими или нестабильными (например, `pscyr`).

## 6. Цели и преимущества

- **Автоматическая валидация**: Стандартизация позволяет автоматически проверять статьи на соответствие требованиям.
- **Долгосрочное хранение**: Структурированный подход обеспечивает сохранение научного смысла публикации на долгие годы.
- **Снижение ручной работы**: Уменьшается объем ручной редакторской правки, связанной с форматированием.
- **Мультиформатность**: Чистый и семантический LaTeX-код является идеальной основой для генерации JATS XML, HTML и ePub.
